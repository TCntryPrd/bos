Connect WhatsApp & Meta with the Facebook Graph API

A complete, step-by-step walkthrough to create a Meta/Facebook app, get a permanent WhatsApp Business access token, and wire it into your BOS Social Media agent so it can read and send messages across WhatsApp, Messenger, and Instagram.

Time: ~30–45 minutes. You need: a Facebook account, a phone number that can receive an SMS/call for WhatsApp, and a credit/debit card on file in Meta (no charge for the free tier — Meta just requires one to enable messaging).

0 · How the pieces fit together

The Meta "Graph API" is one API that covers WhatsApp, Messenger, and Instagram. To use it you create a Meta app (a container), add the WhatsApp product to it, and generate an access token that BOS uses to call the API. Messages coming in arrive through a webhook (a URL on your BOS server that Meta calls).

Create a Meta app

  1. Go to developers.facebook.com/apps and log in. If this is your first time, accept the developer terms.
  2. Click Create App.
  3. For "What do you want your app to do?" choose Other → Next.
  4. App type: choose Business → Next.
  5. Name it something clear (e.g. YourBrand BOS Messaging), enter your contact email, and select or create a Business portfolio (Meta Business account). Click Create app and re-enter your password if asked.
If you don't have a Business portfolio yet, Meta will offer to create one — do it. WhatsApp API requires the app to be tied to a business portfolio.

Add WhatsApp & grab your IDs

  1. On the app dashboard, find the WhatsApp product card and click Set up.
  2. Pick your Business portfolio if prompted. Meta auto-creates a test WhatsApp Business Account and a free test phone number so you can try it before registering your real number.
  3. Open WhatsApp → API Setup in the left menu. Copy and save these three values:
    • Phone number ID (under the test number)
    • WhatsApp Business Account ID (the WABA ID)
    • App ID (top of the page / Settings → Basic)
  4. Also from Settings → Basic, copy your App Secret (click "Show").
Keep the App Secret and tokens private. Anyone with them can send messages as you. Never paste them into a public page, screenshot, or chat.

Get a PERMANENT access token (System User)

The token shown on the API Setup page is temporary (24 hours). For BOS to keep working you need a permanent token, created through a System User.

  1. Go to business.facebook.com/settings (Business Settings).
  2. Left menu: Users → System users → Add. Name it bos-system-user, role Admin, create.
  3. Select the system user → Add assets → Apps → pick your app → toggle Manage app (Full control) → Save.
  4. Still on the system user, click Generate new token. Choose your app. Token expiration: Never.
  5. Under permissions, check whatsapp_business_messaging and whatsapp_business_management (and business_management if shown). Generate.
  6. Copy this token now and save it — Meta only shows it once. This is your permanent Access Token.
This permanent token is the single most important value. It's what you paste into BOS so the Social Media agent can send and read WhatsApp messages indefinitely.

Register your real business phone number

The free test number can only message 5 pre-verified recipients. To message anyone, add your own number.

  1. WhatsApp → API Setup → under "From", click Add phone number.
  2. Enter your business display name, category, and the phone number (it must not already be on a personal WhatsApp account — or first delete WhatsApp from that number).
  3. Verify via SMS or voice call (enter the 6-digit code).
  4. Set a two-step PIN when prompted — save it; you'll need it if you re-register.
  5. Copy the new number's Phone number ID (it replaces the test one in BOS).
New WhatsApp Business numbers start in a free unverified tier with a daily messaging limit (1,000 business-initiated conversations). It scales up automatically as you message more. Submitting Business Verification in Business Settings lifts limits faster.

Wire the webhook so BOS RECEIVES messages

Outgoing messages just need the token. To receive incoming WhatsApp messages, Meta needs to call a URL on your BOS server (a "webhook").

  1. In BOS, open Connectors → WhatsApp (Meta) and copy the two values BOS shows you: your Callback URL (looks like https://your-bos-domain/api/connectors/whatsapp/webhook) and a Verify Token (a random password BOS generated — or type your own).
  2. Back in the Meta app: WhatsApp → Configuration → Webhooks → Edit.
  3. Paste the Callback URL and the Verify Token → Verify and save. Meta sends a one-time GET; BOS answers it automatically. You'll see a green check.
  4. Click Manage on webhook fields and Subscribe to messages (and message_template_status_update if you'll use templates).
The webhook only works once your BOS domain is reachable over HTTPS (it is — every BOS install runs behind Let's Encrypt). If "Verify and save" fails, confirm the Verify Token matches exactly and there are no trailing spaces.

Send your first message & test

You can test straight from a terminal (or let the BOS agent do it). Replace the bracketed values:

curl -X POST \
  "https://graph.facebook.com/v21.0/<PHONE_NUMBER_ID>/messages" \
  -H "Authorization: Bearer <ACCESS_TOKEN>" \
  -H "Content-Type: application/json" \
  -d '{
    "messaging_product": "whatsapp",
    "to": "<RECIPIENT_PHONE_E164, e.g. 15551234567>",
    "type": "template",
    "template": { "name": "hello_world", "language": { "code": "en_US" } }
  }'

A successful call returns a JSON with a messages[0].id. hello_world is a pre-approved template every new account has. After the recipient replies, you have an open 24-hour window in which you can send free-form text:

-d '{ "messaging_product":"whatsapp", "to":"<NUMBER>",
      "type":"text", "text":{ "body":"Hi! This is your BOS agent." } }'
Key rule: outside the 24-hour window you can only send pre-approved message templates. Inside it (after the customer messages you), free text is fine. Your Content Creation agent can draft templates; submit them in WhatsApp → Manage templates.

Extend to Messenger & Instagram (same app)

Because it's all the Graph API, you reuse the same app:

  • Messenger: add the Messenger product → connect a Facebook Page → generate a Page access token → subscribe the same webhook to messages.
  • Instagram: add Instagram → connect a Professional/Business IG account that's linked to that Facebook Page → request instagram_basic + instagram_manage_messages.
  • To post content (not just message), request pages_manage_posts and use the Page's /feed endpoint; for IG, the content-publishing endpoints.
Going beyond messaging your own accounts (e.g. managing clients' pages) requires Meta App Review for the advanced permissions. For your own brand, the standard access above is enough.

Paste it all into BOS

Open Connectors → WhatsApp (Meta) in BOS and fill in:

Access TokenThe permanent System User token (Step 3)

WhatsApp Business Account IDWABA ID (Step 2)

Save. Your Social Media agent now reads incoming WhatsApp messages from the dashboard and can reply, draft, and schedule across your connected Meta platforms.

Done. Ask your Social Media agent: "Send a WhatsApp test message to my number and confirm it went through."

Troubleshooting

"Verify and save" on the webhook fails

The Verify Token must match exactly (case-sensitive, no spaces). Confirm your BOS domain loads over https:// in a browser first.

Token expired after a day

You used the temporary token from the API Setup page. Redo Step 3 with a System User and expiration Never.

"(#131030) Recipient phone number not in allowed list"

You're still on the test number, which only messages pre-added recipients. Add the recipient under the test number's "To" list, or register your real number (Step 4).

"(#100) Param messaging_product is required"

Every request body must include "messaging_product": "whatsapp".

Messages send but you receive nothing

The webhook isn't subscribed to messages. Step 5, "Manage" → Subscribe.
